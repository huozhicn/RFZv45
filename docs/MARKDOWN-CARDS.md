# Markdown 交互卡片设计

## 目标

对话消息渲染 Markdown，Agent 回复带交互按钮（确认/取消/修改），用户直接在卡片上操作。

## Markdown 渲染

ChatPanel 内置轻量渲染器，支持：

| 语法 | 用途 |
|------|------|
| `**粗体**` `*斜体*` | 强调 |
| `\`代码\`` | 字段名/值 |
| `- 列表` | 枚举 |
| `` ```text ``` `` | 代码块 |
| `| 表头 |` | 表格（Agent 最常用） |
| `[确认](action:confirm:0)` | 按钮链接（特殊语法） |

## 交互卡片

Agent 在 `response` 里用 Markdown 组织数据，`actions` 数组带确认/取消：

```json
{
  "response": "已准备新建客户：\n\n| 字段 | 值 |\n|------|-----|\n| 姓名 | 张三 |\n| 电话 | 13800001111 |\n| 类型 | 居士 |\n\n确认创建？",
  "actions": [
    { "type": "confirm", "message": "创建客户张三", "on_confirm": { "sql": "CREATE customer CONTENT { name: '张三', ... }" } },
    { "type": "reply", "text": "取消" }
  ]
}
```

前端渲染流程：

```
1. Markdown → HTML（表格 + 文本）
2. actions 中 confirm → 卡片底部追加按钮条
3. 用户点「确认」→ execConfirm(sql) → 刷新表格
4. 用户点「取消」→ 消息折叠或标记已处理
```

## 点击数据行 → 底部抽屉

不再从右侧滑出（会遮盖对话面板）。改为：

- 点击行 → 底部 60vh 抽屉滑出
- 主区域表格可滚动
- 抽屉内：字段列表 + 保存/删除按钮

布局：

```
┌──────────┬──────────────────────┬──────────────┐
│  Sidebar │   Table (可滚)       │   ChatPanel  │
│          │                      │   messages   │
│          ├──────────────────────┤              │
│          │  Detail Drawer       │   input      │
│          │  (底部 60vh 弹出)     │              │
└──────────┴──────────────────────┴──────────────┘
```

## 去掉新建按钮

Agent 对话取代手动新建。「新建客户张三」→ Agent 预填 → 用户确认 → 创建。SchemaTable 顶部只保留搜索和翻页。
